Metodología

Un pipeline completo de ciencia de datos

Desde la extracción de los datos hasta el despliegue de una API REST funcional, el proyecto cubre todas las fases del ciclo de vida de un modelo de Machine Learning.

01
Scraping & Dataset
Extracción de 11.826 anuncios de Idealista.com con datos de precio, localización, superficie, habitaciones, baños y características adicionales.
Pandas · CSV
02
Limpieza de datos
Eliminación de duplicados, tratamiento de outliers (pisos con más de 10 habitaciones), imputación de valores nulos con la mediana y filtrado de precios inválidos.
Numpy · Seaborn
03
Feature Engineering
Creación de 5 nuevas variables derivadas: zona_codificada (Target Encoding), extras desde texto libre (NLP), metros_por_hab y ratio_banos.
Target Encoder
04
Entrenamiento XGBoost
Modelo de Gradient Boosting entrenado con división 80/20 train-test. Selección del algoritmo tras comparativa frente a Random Forest y regresión lineal.
XGBoost · Sklearn
05
Evaluación
Evaluación con R² Score, MAE y análisis de importancia de variables. Gráfico de dispersión precios reales vs. predicciones para validación visual.
Sklearn Metrics
06
API & Despliegue
Serialización del modelo con joblib y exposición mediante una API REST construida con FastAPI + Uvicorn. Frontend en HTML/CSS/JS puro.
FastAPI · Joblib

Gradient Boosting:
ensembles que aprenden de sus errores

XGBoost construye árboles de decisión de forma secuencial: cada nuevo árbol se enfoca en corregir los errores del anterior. El resultado es un ensemble altamente preciso que supera a modelos lineales y redes neuronales poco profundas en datasets de tamaño medio con características heterogéneas como el nuestro.

Se añadió regularización L1 y L2 para evitar overfitting, algo crítico cuando la zona (un factor dominante) puede sesgar el modelo hacia los distritos más caros.
